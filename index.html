Reflexión de mi camino como desarrollador

Mi carrera como desarrollador no comenzó con Java. Mis cimientos se forjaron en el mundo del software embebido automotriz gracias a mis estudios como Ingeniero Electrónico. Durante más de cuatro años, mi lenguaje principal fue C, mi biblia fue el estándar MISRA, y mi realidad diaria fue trabajar con microcontroladores, AUTOSAR y protocolos de bajo nivel.

Este mundo me enseñó la disciplina. Me enseñó que cada byte de memoria cuenta y que un error de software tiene consecuencias en el mundo real.

Decidir saltar a Java Fullstack con Generation Mexico fue un giro deliberado. Quería pasar 'de los circuitos a la nube'. El "choque" inicial fue real: pasar del manejo manual de memoria en C a la recolección de basura de Java, y de la configuración de stacks de BSW a la arquitectura de una API REST con Spring Boot, es cambiar de paradigma.

Aquí es donde descubrí mi ventaja única: soy un desarrollador Java que entiende fundamentalmente lo que ocurre en el hardware. Mi experiencia en embebidos me da una perspectiva única sobre la eficiencia y la robustez del código.

Áreas de Mejora: Soy consciente de que el ecosistema web es demasiado extenso y busco el poder tener un conocimiento amplio.

Metas: Mi meta a corto plazo es clara: asegurar mi primer rol como Java Fullstack Developer y aplicar este nuevo conocimiento en un entorno empresarial. A largo plazo, mi objetivo es la convergencia. No quiero abandonar mi experiencia en embebidos; quiero fusionarla. Mi sueño es trabajar en el nexo del hardware y el software: en el mundo del IoT (Internet of Things) o en la próxima generación de vehículos conectados, siendo el 'puente' que puede hablar con el microcontrolador en C y, en la misma reunión, diseñar la API REST en Java que consumirá sus datos.
